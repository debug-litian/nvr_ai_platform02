### 2d. core/preview_thread.py（`d:/nvr_ai_platform/core/preview_thread.py`）和 core/stream_capture.py（`d:/nvr_ai_platform/core/stream_capture.py`）

## 3. RTSP 流的配置方式

流是**逐一**配置的——确切地说，是一共有 1 个流：

- 默认值硬编码在 `settings.RTSP_URL` 中（`rtsp://admin:111111..@192.168.124.2:554/Preview_10_main`）
- GUI 中有一个 `QLineEdit`（`self.rtsp_input`），预填了默认 URL
- 用户可以在点击“开始预览”之前编辑 URL
- 点击“开始预览”后，主窗口将 URL 传递给单个 `PreviewThread`
- 不支持批量添加、批量编辑或多 URL 配置
- 不支持从配置文件或数据库加载通道

流的启动流程：
1. 点击“▶ 开始预览”
2. 从 `QLineEdit` 读取 URL
3. 创建 `PreviewThread(url)`
4. 连接信号（`frame_ready`、`status_updated`、`fps_updated`）
5. 启动线程，线程调用 `_open_capture()` -&gt; `cv2.VideoCapture(url)` 或 ffmpeg 子进程
6. 启动检测处理线程
7. 启动 15 FPS 的 `QTimer` 用于显示刷新

## 6. 当前多通道支持架构总结

**本质上没有多通道架构。** 当前系统是单数据流水线：

```
┌─────────────┐     ┌────────────────┐     ┌──────────────┐
│ 单个 RTSP   │ --&gt; │ PreviewThread  │ --&gt; │ MainWindow   │
│ URL 输入    │     │ (QThread,      │     │ (主线程)     │
│ (QLineEdit) │     │  拉流+解码)    │     │              │
└─────────────┘     └────────────────┘     │ - QTimer     │
                                           │   (15fps)    │
                                           │       ↓      │
                                           │ VideoWidget  │
                                           │ (单个组件)   │
                                           └──────────────┘
                                                  │
                    ┌─────────────────────────────┘
                    │
                    ▼
           ┌──────────────┐
           │ 检测模型      │ (后台守护线程)
           │ - 绿线检测    │
           │ - YOLO        │
           │ - 实时索引    │
           └──────────────┘
```

要添加多通道支持，需要：
1. 在 `settings.py` 中添加通道/URL 列表配置（例如，`CHANNELS = [...]`）
2. 维护 `PreviewThread` 或 `StreamCapture` 实例数组
3. 在 GUI 中添加 `QGridLayout`，可以动态创建 `VideoWidget` 实例
4. 更新 `_refresh_display()` 以处理每通道的帧
5. 潜在地将检测流水线复制或复用至多通道

### 4. core/index_thread.py -- 索引线程架构

**文件**: `d:\nvr_ai_platform\core\index_thread.py`

**模式**: 与搜索线程相同——按请求创建线程。一次索引一个视频文件。

**流程**:
1. 通过 `cv2.VideoCapture` 打开视频。
2. 根据 `sample_fps` 计算帧间隔。
3. 加载 CLIP（它自己的实例）。
4. 创建 FAISS `IndexFlatL2` (512维)。
5. 循环遍历帧：按 `interval` 跳过，BGR-&gt;RGB-&gt;PIL-&gt;CLIP 编码，追加向量 + 元数据。
6. 将索引保存到 `settings.INDEX_FILE`，将映射保存到 `settings.MAPPING_FILE`。

### 5. detectors/yolo_detector.py -- YOLO 初始化、单例和批处理

**文件**: `d:\nvr_ai_platform\detectors\yolo_detector.py`

### 6. detectors/green_line_detector.py -- 绿线检测器

**文件**: `d:\nvr_ai_platform\detectors\green_line_detector.py`

纯函数 `detect_green_and_vertical_lines(frame, threshold)` -- **无状态**，无类实例。

**算法**:
1. BGR -&gt; HSV 转换。
2. 为绿色范围 (H:35-85, S:50-255, V:50-255) 创建掩码。
3. 计算绿色像素比例。
4. 在掩码上运行 Canny 边缘检测 + Hough 直线检测。
5. 过滤垂直直线（|dx| &lt; 0.3 * |dy|）。
6. 如果 `green_ratio &gt; threshold` **或** 存在任何垂直线，则返回 `abnormal=True`。

**多通道相关性**: 高度可重用。它是无状态的，只是对帧进行操作。按帧调用，开销很小（主要是 HSV 转换 + Canny + Hough——几个毫秒）。对多通道无需更改。

### 7. detectors/false_positive_filter.py -- 误报过滤器

**文件**: `d:\nvr_ai_platform\detectors\false_positive_filter.py`

**有状态的类** — 维护检测历史以进行运动一致性检查。不是一个单例，但它用于每个 `MainWindow` 实例。

**算法**:
1. **基于置信度的过滤**: 移除 `conf &lt; conf_threshold` 的检测。
2. **基于面积的过滤**: 移除面积 &lt; `area_threshold`（默认1000平方像素）的框。
3. **运动一致性**（第34-53行）: 将当前检测与前一帧的检测进行比较。只有中心在先前检测的50px内的检测才通过。这消除了瞬态误报，但可能会漏掉快速移动的物体或场景切换。

**多通道相关性**: **有状态，是每个通道的**。历史按通道维护。如果你有多个通道，每个需要它自己的 `FalsePositiveFilter` 实例。当前设计将过滤器的历史链接到单个摄像头流；你不能共享它。

## 2. 通道 / 多路流支持 —— 当前限制

**当前仅支持 1（一个）RTSP 通道。** 没有多通道支持。以下是支撑这一结论的证据：

### 2a. config/settings.py（`d:/nvr_ai_platform/config/settings.py`）

只有一个单一的 RTSP URL：

```python
RTSP_URL = "rtsp://admin:111111..@192.168.124.2:554/Preview_10_main"
```

没有通道列表、没有 RTSP URL 字典/数组、没有通道配置结构、没有网格布局配置。与多通道相关的设置或标签（`channels`、`grid`、`rows`、`cols`、`cameras`）均为零。

### 2b. gui/main_window.py（`d:/nvr_ai_platform/gui/main_window.py`）

- 创建**一个** `VideoWidget` 实例：`self.video = VideoWidget(self)`
- 创建**一个** `PreviewThread` 实例：`self._preview_thread = PreviewThread(url, parent=self)`
- 创建**一行** RTSP 输入组件：`self.rtsp_input = QLineEdit(self)`
- 无网格布局（`QGridLayout`）、无通道数迭代、无多组件映射。
- 布局是垂直的：单个视频组件 -&gt; RTSP 控制栏 -&gt; 搜索栏 -&gt; 进度条 -&gt; 结果表格 -&gt; 底部区域（日志 + 告警 + 运行时间）。

### 2c. gui/video_widget.py（`d:/nvr_ai_platform/gui/video_widget.py`）

一个独立的单帧显示组件。每个实例持有：
- 一张 `_qimg`（QImage）
- 一组叠加信息字段
- 缩放/平移状态
