## 1. YOLO 画框开关（调试用）

在 RTSP 控制栏右侧新增了一个 **「YOLO框」复选框**：

- **勾选**：画面上绘制检测框（默认开启）
- **取消勾选**：画面显示原始画面，不画任何框，但后台检测仍然运行，日志继续输出

这对于调试绿线问题很有用——关掉框后你能看到「纯净」的原始画面，判断画面本身有没有绿线。

修改的文件：
- [config/settings.py:43](config/settings.py#L43) — 新增 `SHOW_YOLO_BOXES = True` 配置项
- [gui/main_window.py:148-152](gui/main_window.py#L148-L152) — UI 新增 `QCheckBox` 开关
- [gui/main_window.py:386-392](gui/main_window.py#L386-L392) — `_refresh_display` 根据开关决定是否画框

## 2. 绿线问题的本质

绿线不是 YOLO 画框的颜色导致的。看一下代码流程：

1. 后台线程做 YOLO 检测 + 绿线检测是**并行的**（都在 `_process_frame` 里）
2. 画框只在**显示用**的 frame copy 上（[main_window.py:390-392](gui/main_window.py#L390-L392)），原始帧不受影响
3. 绿线检测器 `detect_green_and_vertical_lines` 检查的是**原始帧**中的绿色像素比例

所以绿线的本质原因可能是：
- **摄像头/编码器问题**：RTSP 流本身产生了绿条（常见于海康/大华摄像头在丢包或带宽不足时）
- **解码器问题**：ffmpeg 解码时的花屏/绿条
- **传输丢包**：网络不稳定导致部分宏块损坏，显示为绿色

你现在可以用开关关掉 YOLO 框，观察画面确认绿线是来自解码/传输层还是画框产生的视觉干扰。

## 3. YOLO 框颜色自定义

**完全可以**，而且我已经帮你改了。原来全部是绿色 `(0, 255, 0)`，现在 [main_window.py:504-523](gui/main_window.py#L504-L523) 定义了 16 个 COCO 类别各自的颜色：

| 类别 | 颜色 | 视觉效果 |
|------|------|---------|
| person | 绿色 | 🟢 |
| car | 橙色 | 🟠 |
| truck | 紫色 | 🟣 |
| bus | 黄色 | 🟡 |
| dog | 红色 | 🔴 |
| cat | 蓝色 | 🔵 |
| ... | ... | ... |

标签文字也从黑色改成了**白色**，在彩色背景上更容易看清。如果你想自己调颜色，直接改 [gui/main_window.py:504-523](gui/main_window.py#L504-L523) 里的 `_CLASS_COLORS` 字典就行，值是 `(B, G, R)` 格式（OpenCV 是 BGR 而不是 RGB）。
